## 2. 背景与问题

0-3 岁宝宝的成长发育信息分散在 PDF 指南、表格、体检建议和疫苗计划中，普通家长很难快速按宝宝当前月龄找到可执行的参考内容。当前 H5 需要把结构化数据、原文依据和轻量交互整合成一个可查询、可核对、可反馈、可分享的移动端工具。

## 3. 产品目标

### 3.1 用户目标

- 家长可以输入或选择宝宝出生日期，自动匹配当前月龄阶段。
- 家长可以按月龄查看体重、身高、头围等生长指标参考范围。
- 家长可以按维度查看睡眠、辅食、语言、动作、认知、情绪社会性、疫苗、体检、警示信号等内容。
- 家长可以快速搜索关键词，并定位到相关阶段、维度和原文依据。
- 家长可以查看数据来源、版本说明和免责声明，判断信息可信度。
- 家长发现错误时可以提交反馈，形成数据修正闭环。

### 3.2 业务目标

- 把 0-3 岁育儿参考内容从静态资料转成可浏览、可搜索、可校对的 H5 产品。
- 用原文依据和数据说明降低内容可信度疑虑。
- 通过成长卡和分享海报形成轻量传播入口。
- 用反馈入口收集错误数据、文字问题和来源问题，支持后续数据迭代。

### 3.3 非目标

- 不提供医疗诊断、治疗建议或个体化处方。
- 不替代线下儿保、疫苗接种单位或医生判断。
- 首版不要求完整账号体系；宝宝信息可优先本地保存。

## 4. 用户与场景

| 用户 | 典型场景 | 关键诉求 |
| --- | --- | --- |
| 新生儿及婴幼儿家长 | 想知道当前月龄体重、身高、睡眠、喂养是否在参考范围内 | 快速定位当前阶段，信息表达清晰 |
| 照护者 | 日常照护时查询辅食、睡眠、动作发展、警示信号 | 按维度浏览，内容可读、可筛选 |
| 需要核对信息的用户 | 发现页面数据和 PDF 原文可能不一致 | 查看原文依据，一键反馈错误 |
| 分享型用户 | 想把宝宝当前阶段信息分享给家人 | 生成成长卡或分享海报 |

## 6. 核心流程

### 6.1 首次访问与阶段匹配

1. 用户进入首页。
2. 系统读取本地宝宝信息；若无信息，默认展示 0-3 岁成长发育首页和阶段选择。
3. 用户填写出生日期、性别等信息。
4. 系统计算宝宝当前年龄，例如 `10个月5天`。
5. 系统匹配到对应阶段，例如 `10-12个月`。
6. 首页展示该阶段生长指标、喂养、发育、体检、疫苗等重点内容。

### 6.2 按阶段查看

1. 用户在首页切换月龄阶段。
2. 页面刷新对应阶段的生长指标卡片。
3. 用户进入阶段详情。
4. 阶段详情按生长、喂养、发育、医疗保健分组展示。
5. 用户可展开/收起分组，查看来源页标记。

### 6.3 按维度查看

1. 用户进入维度中心。
2. 选择体重、身高、头围、睡眠、辅食、疫苗、体检等维度。
3. 生长指标类页面展示按月龄和性别拆分的范围表。
4. 文本类维度展示时间线或卡片列表。
5. 用户可进入原文依据核对数据来源。

### 6.4 搜索与反馈

1. 用户进入搜索页，输入关键词或点击热门搜索。
2. 搜索结果按内容、维度、阶段分类展示。
3. 结果为空时展示低数据说明和跳转建议。
4. 用户发现错误时进入反馈表单。
5. 反馈提交后展示反馈编号和后续处理说明。

### 6.5 分享与数据更新

1. 用户在阶段详情生成成长卡。
2. 用户可复制摘要、生成海报、保存图片、微信分享或复制链接。
3. 当本地数据版本低于最新版本时，展示数据更新提示。
4. 用户可选择稍后或立即更新数据。

## 8. 内容与数据需求

### 8.1 数据分类

| 分类 | 示例字段 |
| --- | --- |
| 阶段 | 阶段 ID、起止月龄、阶段标题、阶段说明 |
| 生长指标 | 阶段、性别、体重范围、身高范围、头围范围、单位、来源页 |
| 营养喂养 | 辅食添加、乳类营养、营养剂、来源页 |
| 发育表现 | 动作、语言、认知、情绪社会性、来源页 |
| 日常照护 | 睡眠、排便、乳类喂养、来源页 |
| 医疗保健 | 疫苗、体检、警示信号、来源页 |
| 来源信息 | PDF 名称、页码、数据表、更新时间 |
| 反馈 | 问题字段、问题类型、描述、联系方式、反馈编号、状态 |

### 8.2 数据规则

- 男孩和女孩指标需分色展示。
- 4-6 月、7-9 月等只有增长速度的数据，不能误展示为具体体重范围。
- 缺失数据需明确说明原因，并给出可替代查看路径。
- 所有可反馈字段应保留来源页和数据表信息。
- 数据版本应可追踪，更新后需要刷新缓存或提示用户重新加载。

## 9. 状态与异常

| 状态 | 展示要求 |
| --- | --- |
| 加载中 | 使用骨架屏或加载提示，避免页面空白 |
| 加载失败 | 展示失败原因、重试按钮和返回首页入口 |
| 空数据 | 说明当前筛选下数据较少，提供替代操作 |
| 原文缺失 | 展示来源缺失说明，并引导反馈 |
| 反馈提交失败 | 保留用户输入，支持重新提交 |
| 数据更新失败 | 保持旧版本可用，提示稍后重试 |

## 10. 非功能需求

- 移动端首屏加载需尽量轻量，核心内容优先展示。
- 页面宽度按 H5 设计适配，需兼容主流 iOS/Android WebView。
- 数据说明和免责声明必须在首页、数据说明页和关键详情页可见或可达。
- 搜索、反馈和分享操作需有明确成功/失败反馈。
- 本地保存宝宝信息时需避免上传敏感信息，除非用户主动提交反馈。
- 所有图片、文档和页面引用优先使用相对路径，便于项目内迁移。

## 12. 验收清单

### 12.1 P0 验收

- 首页可按月龄展示体重、身高、头围核心指标。
- 阶段详情可展示当前阶段完整分组内容。
- 维度中心可进入生长指标、睡眠、疫苗、体检等详情。
- 搜索首页和搜索结果页可正常使用，并覆盖空状态。
- 原文依据页能展示来源页、字段、PDF 摘录和反馈入口。
- 数据说明页包含来源、版本、更新时间、适用范围和免责声明。
- 反馈表单可提交并能带入当前错误字段上下文。

### 12.2 P1 验收

- 宝宝信息保存后首页能展示个性化内容。
- 生长指标详情支持筛选和图表说明。
- 疫苗详情、体检详情、成长卡和数据更新页可用。
- 反馈成功页能展示反馈编号。
- 加载失败、空数据、低数据说明等状态完整。

### 12.3 P2 验收

- 收藏、反馈记录、分享海报保存、微信分享、复制链接等增强能力可用。
- 成长卡内容适合截图或转发。
- 最近查看和历史记录可清理。
